# 18 工具链链接索引与模块替换清单

> 用途：这是 Sidebar Knowledge Agent 项目的“工具链索引文档”。  
> 它不是总体架构文档，而是给 AI Agent / 开发者 / 外部帮手用来快速定位可参考项目、npm 包、官方 API 和可替换模块的链接清单。  
> 目标是：开发时不用再一个个搜索，直接按模块打开对应链接。

---

## 0. 使用方式

建议放到：

```text
docs/18_工具链链接索引与模块替换清单.md
```

给 AI Agent 的使用指令：

```text
请先阅读 docs/18_工具链链接索引与模块替换清单.md。
当你实现某个模块时，只查看该模块下列出的项目和文档。
不要盲目复制完整项目。
优先抽取接口、工具协议、解析逻辑和权限设计。
复制任何代码前必须检查 License，并在 NOTICE.md 记录来源。
```

---

## 1. 模块总览表

| 模块 | 优先参考 | 主要用途 | 推荐处理方式 |
|---|---|---|---|
| Agent Runtime | OpenCode | agent loop / provider / tool call / permission | 抽思想或局部代码 |
| 浏览器插件框架 | WXT / Plasmo / Chrome 官方文档 | MV3 插件、Side Panel、content script | 优先官方 API，框架可选 |
| 网页转 Markdown | Readability / Turndown / MarkDownload | 正文提取、HTML 转 MD | 直接依赖 npm + 参考插件流程 |
| 网页完整保存 | SingleFile | 完整页面资产保存思路 | 只参考，不建议直接复制 |
| 视频字幕 | youtube-transcript-api / YouTube.js / B站字幕工具 | YouTube/Bilibili 字幕获取 | 先字幕，后音频转写 |
| 媒体处理 | ffmpeg / yt-dlp / whisper.cpp / faster-whisper | 音频提取、可选下载、ASR | 工具封装，不给 LLM shell |
| 资源扫描 | Media-downloader / Chrome DOM/performance API | 图片、PDF、媒体链接扫描 | 参考资源分类和 UI |
| Vault / Markdown | gray-matter / Obsidian 生态 | frontmatter、Markdown 管理 | 自己实现 save_note |
| 搜索索引 | ripgrep / SQLite FTS5 / LanceDB | 全文搜索、向量检索 | MVP 先全文，后向量 |
| MCP | MCP TS SDK / MCP servers | Claude Code / Claude Desktop 共享知识库 | 只读优先 |

## 2. Agent Runtime / Harness 模块

### 2.1 OpenCode

- 链接：https://github.com/anomalyco/opencode
- 官网：https://opencode.ai/
- 用途：参考 agent runtime、模型 provider、工具调用循环、权限模式、session。
- License：MIT，复制代码必须保留 License / copyright notice。

#### 可借鉴部分

```text
1. model provider 适配层
2. agent loop
3. tool call 执行流程
4. permission mode
5. session / conversation 存储
6. agent mode 设计
7. 配置系统
```

#### 不建议直接搬的部分

```text
1. LSP
2. Git diff / patch
3. coding prompt
4. 测试运行器
5. IDE 集成
6. PR / issue 工作流
```

#### 替换关系

| OpenCode 概念 | 本项目替换 |
|---|---|
| build agent | curator / media / resource / librarian |
| plan agent | reader |
| code edit tool | save_markdown_note |
| bash tool | 受控工具层 |
| repo scan | vault index |
| file edit | note update |
| coding session | capture task session |

#### 给 AI 的指令

```text
只分析 OpenCode 的 provider、agent loop、tool call、permission、session。
不要迁移 coding prompt、LSP、Git patch、IDE 集成。
把 OpenCode 的 build/plan 模式替换成 reader/curator/media/resource/librarian。
```

## 18. 最终原则

```text
开源项目 = 查实现、查边界、查工具用法
本项目工具层 = 真正可调用接口
LLM Harness = 决定何时调用哪个工具
Permission Guard = 决定能不能调用
Vault = 所有知识最终落点
MCP = Claude Code / Claude Desktop 的共享入口
```

不要把“参考项目”直接变成“LLM 自由调用的能力”。  
必须先封装成工具，再注册到 Runtime。
