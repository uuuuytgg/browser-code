# Skill 路由约束规则

## 核心原则：资讯类查询必须先问用户

当用户查询 AI 资讯 / 行业动态 / 热点新闻（包括但不限于"世界模型"、"AI 圈"、"今天有什么新闻"等）时：

### 1. aihot skill 不得自动执行

- 即使查询内容匹配 aihot skill 的触发词（"AI 圈"、"AI 新闻"、"AI 日报"等），**也不得直接调用 aihot 的 API**
- 必须先向用户展示可用的信息管道选项，**等用户确认后再执行**

### 2. ProReader 优先

- 对于模糊研究请求（fuzzy research request），**优先等待 ProReader 的路由结果**
- 在用户确认之前，不得执行任何 skill 的工作流

### 3. 禁止并行撒网

- 不要在拿到任何结果之前就并行发出去所有搜索工具
- 正确的流程：先确认路径 → 用一个管道拿到结果 → 根据结果质量判断是否需要补充搜索
- 避免冗余的重复搜索（比如已有 AI HOT 的成品数据就不要再去 B站搜一遍相同的关键词）

### 4. 可选管道清单（展示给用户选择）

当用户查 AI 资讯时，应该展示以下选项让用户决定：

```
a) **AI HOT API** — 成品 AI 资讯库，覆盖几十个信源，中文摘要现成
   ✅ 省时、结构好、覆盖面广
   ⚠️ 仅限 AI 行业，最近的才准

b) **ProReader + MCP 工具链**（B站/抖音/小红书 + 网页搜索）
   ✅ 可查全网任何内容，不限于 AI
   ⚠️ 每次实时搜索，质量取决于平台

c) **两者都跑** — AI HOT 出全景 + MCP 补具体细节
```

### 5. 异常处理：aihot API 不可用

- 如果 AI HOT API 返回 403 或空结果，**告知用户后**再 fallback 到其他工具
- 不要在用户不知情的情况下切换信息源

## 适用范围

本约束文件适用于所有涉及 AI 资讯 / 行业新闻查询的场景。其他类型的任务（如代码生成、文件操作等）不受此约束。
