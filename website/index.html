II. 关于 / 起源 • Browser‑Code / Volume 01 002 / 008
关于这个 Agent · Nº 02

它不是一台独立服务器，也不是一个浏览器插件——它就是此刻正在跟你对话的这个 Agent.

Browser‑Code 基于 OpenCode fork 定制。OpenCode 提供了完整的 agent 运行时——会话管理、provider 抽象、工具注册、权限系统——Browser‑Code 在此之上增加了一套面向「知识捕获与管理」的工具包。入口点 bin/browser-code.cjs 委托给 OpenCode 编译后的二进制。

III. 能力 · 工具注册表 • 9 工具 / 3 风险等级 003 / 008
BROWSER‑CODE  ·  工具注册表  ·  BC/01
能力矩阵 · Nº 03

9 个注册工具，按风险等级分层.

每个工具有完整 Schema、风险评级和允许模式。不允许的指令：run_shell、execute_command、eval_js、run_python。

01Low Risk

web_to_markdown

网页转 Markdown，基于 Mozilla Readability + Turndown，保留 GFM 语法。

02Medium Risk

save_markdown_note

保存笔记到本地 Vault，自动生成 YAML frontmatter + 受控词表标签。

03Low Risk

search_vault

全文检索 Vault 内容，SQLite FTS5 引擎，中文降级 LIKE 模糊查询。

04High Risk

ffmpeg_extract_audio

提取视频音轨，需用户确认。搭配 Doubao ASR 生成字幕。

V. 方法 / 知识管线 • 4 阶段 · 全程本地 005 / 008

从捕获到服务.

+

用户输入 → OpenCode Agent Runtime → 工具注册表 → Markdown Vault → MCP 知识服务。每个阶段都迭代、可视、文件驱动。

01

捕获→

web_to_markdown 抓取网页、fetch_transcript 拉字幕、scan_page_resources 扫资源——内容入口归一。

02

组织→

save_markdown_note 写入 Vault，自动生成 YAML frontmatter，受控词表标签不超过 5 个。

03

索引→

build_index 构建 SQLite FTS5 全文索引。中文不足时降级 LIKE 模糊查询。

04

服务

通过 @ska/mcp-server 对外暴露 MCP 知识服务，任何 MCP 客户端均可消费 Vault 内容。

文件驱动。每个阶段产物可查。 github.com/opencode-ai/opencode  MIT
